# MdStory 写作技能

MdStory 是基于 Markdown 和 Handlebars 的互动小说脚本格式。本技能指导你使用 MdStory 格式创作互动故事（写在 `.md` 文件中）。

开始写作前先阅读本 skill 目录下的参考示例（`examples/`）以理解格式。

## 快速参考

一次对话中的工作流：

1. **明确故事设定** — 主题、风格、分支结构复杂度
2. **设计故事结构** — 章节划分、关键分支点、结局数量
3. **编写主入口文件** — 包含 metadata、story 级 hooks、`!include` 引入各章节
4. **编写章节和场景** — 每个场景是 `###` 标题下的一个可渲染单元。**分章节编写时，每写完一章立即执行"章节级检查"**
5. **全篇终检** — 所有章节完成后，执行"全篇终检"，确认全篇一致性

## 编写后检查清单

**检查是强制步骤，不得跳过。** 每写完一个章节执行"章节级检查"，全篇完成后执行"全篇终检"。检查结果必须逐条列出，发现问题必须修复后重新检查。

### 章节级检查（每写完一章立即执行）

| #   | 检查项                       | 检查方法                                                                                                                                                                                                                 |
| --- | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 1   | **章节必须有 id**            | 确认 `## 章节名 {#chapter-id}` 有显式 id                                                                                                                                                                                 |
| 2   | **场景必须有 id**            | 确认每个 `### 场景名 {#scene-id}` 有显式 id                                                                                                                                                                              |
| 3   | **章内 nav 可达**            | 本章内所有 `{{#nav "scene"}}` 的目标场景存在于本章                                                                                                                                                                       |
| 4   | **章内 nav 无死链**          | 跨章 nav 声明的目标章节/场景是否已存在或计划中                                                                                                                                                                           |
| 5   | **模板变量已定义**           | 场景中所有位置的变量都必须有定义——包括 `{{var}}` 插值、`{{#if var}}`/`{{#unless var}}` 条件表达式、`{{#each var}}` 迭代源及子表达式中的变量。每个变量必须在 `globals`、`locals()`、`view()` 或 `assets` 中至少有一处定义 |
| 6   | **条件块闭合**               | 每个 `{{#if}}`/`{{#each}}` 有对应的 `{{/if}}`/`{{/each}}`，嵌套层级正确                                                                                                                                                  |
| 7   | **input 类型正确**           | `{{input}}` 的 type 与默认值匹配（string→字符串、number→数字、boolean→true/false）                                                                                                                                       |
| 8   | **跨章变量用 $ 前缀**        | 需要跨章节保留的 `{{input}}` 变量使用了 `$` 前缀                                                                                                                                                                         |
| 9   | **view() 无副作用**          | `view()` 只返回渲染值，不修改 `globals` 或 `locals`                                                                                                                                                                      |
| 10  | **onEnter/onLeave 职责分离** | 进入时所需的状态修改在 `onEnter()`，离开时的结算在 `onLeave()`                                                                                                                                                           |
| 11  | **include 路径存在**         | 本章内所有 `!include("...")` 引用的文件确实存在                                                                                                                                                                          |

### 全篇终检（所有章节完成后执行）

> 全篇终检只关注跨章节问题，不重复章节级检查已覆盖的单项（模板变量定义、条件闭合、input 类型、view() 副作用、onEnter/onLeave 分离、include 路径存在等已在章节级逐章验证）。

| #   | 检查项                 | 检查方法                                                                                                                                                                           |
| --- | ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | **跨章 nav 全可达**    | 遍历所有章节的所有场景，确认每个 `{{#nav "chapter.scene"}}` 的目标真实存在                                                                                                         |
| 2   | **全局变量一致性**     | 同一变量名在全篇中类型一致、语义一致（不会在 A 章是布尔、B 章变成字符串）；不出现所有变量都塞进 `globals` 的滥用                                                                   |
| 3   | **所有场景从入口可达** | 从故事入口场景出发，沿所有 `{{#nav}}` 路径遍历，每个场景都能被至少一条路径到达。不存在无法到达的"死场景"                                                                           |
| 4   | **结局可达**           | 每条故事线最终都能到达至少一个 `{{#nav null}}` 结局                                                                                                                                |
| 5   | **无逻辑死循环**       | 检查 nav 链路：不存在"场景 A → 场景 B → 场景 A"的纯循环路径（除非循环中有 `onEnter`/`onLeave` 修改状态最终能改变分支条件跳出）。如发现无条件循环，必须添加退出路径或状态变化来打破 |
| 6   | **变量初始化完整**     | frontmatter `globals` 和 `globals()` 返回值覆盖了全篇所有直接读取的全局变量                                                                                                        |
| 7   | **章节 id 唯一**       | 所有 `##` 的 id 全篇不重复                                                                                                                                                         |
| 8   | **命名规范统一**       | 全篇变量命名风格一致（不会中英文混用表示同一概念），id 使用简短稳定的英文                                                                                                          |
| 9   | **include 链完整**     | 主入口文件的 `!include` 覆盖了所有章节文件，递归 include 链路完整                                                                                                                  |

### 检查报告格式

每次检查完成后，输出如下格式的报告：

```
## 检查报告：{章节名 或 "全篇终检"}

### 通过项
- [x] 检查项描述
...

### 问题项
- [ ] 检查项描述 → 具体问题：{变量名/nav目标/场景名}
- [ ] 检查项描述 → 具体问题：...

### 修复记录
- 修复了 xxx 处的 yyy 问题
...

**结论：{章节级/全篇} 检查通过，共 N 项通过，M 项已修复。**
```

只有所有问题项都已修复且结论为"通过"后，才能继续编写下一章（或交付故事）。

## 文档结构

```
┌─ metadata (YAML frontmatter)
│   title, globals 静态初始值, assets
│
├─ # 故事标题 (可选)
│   ├─ <style> CSS 样式
│   └─ <script> story 级 hooks
│       globals(), onStart()
│
├─ ## 章节 — 场景的分组单位
│   ├─ <script> chapter 级 hooks
│   │   locals(), onEnter(), onLeave()
│   └─
│       ├─ ### 场景 — 可渲染单元
│       │   ├─ <script> scene 级 hooks
│       │   │   view(), onEnter(), onLeave()
│       │   └─ Handlebars 模板内容
│       │
│       └─ ### 场景 ...
│
└─ ## 章节 ...
```

## 层级规则

| 层级  | 标题     | 作用                                            |
| ----- | -------- | ----------------------------------------------- |
| `#`   | 故事标题 | 可选。章节/场景之前的 `<script>` 输出故事级钩子 |
| `##`  | 章节     | 将场景分组。可拥有自己的钩子和 `locals`         |
| `###` | 场景     | 可渲染单元，包含 Handlebars 模板                |

- 没有 `#` 标题时，故事标题使用 metadata 中的 `title`；都没有则为空。
- 任何 `##` 之前的 `###` 自动归入隐式默认章节。
